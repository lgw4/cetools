## Summary

Build a Cepheus Engine Navy character generator as a data-driven Python library, exposed via a `cetools character generate` Typer CLI subcommand. The core engine accepts a `Career` frozen dataclass and a `DiceRoller` protocol, enabling deterministic testing and future extensibility to additional careers without engine modification.

## Key Design Decisions

### Career Data Interface

All career variation lives in a `Career` frozen dataclass (see [data-model.md](data-model.md)). The generation engine imports `Career` and `DiceRoller` only — no `NavyCareer` references. Adding a new career requires: define a `Career` instance in `careers/<name>.py`, wire it to a CLI command. Zero engine changes.

### RNG Injection

`generate_character(career, roller=DiceRoller())` accepts an injectable roller. Default uses `random.randint`. Tests inject a controlled roller to produce deterministic outcomes for every branch of the generation state machine.

### SRD Corrections

The spec FRs contain values that differ from the SRD. **Implementation follows SRD** (per spec Assumptions). See [research.md](research.md#discrepancies-spec-vs-srd) for the full correction table. Key differences:
- Qualification: Intelligence 6+ (not Endurance 5+)
- Commission: Social Standing 7+ (not Education 6+)
- Advancement: Education 6+ (not Education 7+)
- Rank titles: Starman–Commodore (not Ensign–Fleet Admiral)
- Rank 3 bonus: Tactics-1 (not Leadership-1)

### Background Skills (Simplified)

Per spec Assumption: 3 skills at Level 0 from a default list (not Education-DM-scaled, not homeworld-dependent). Full homeworld generation is deferred.

### Aging Distribution

When aging reduces multiple physical stats, the player distributes reductions. In a CLI-only MVP with no player input mid-generation, reductions are applied in order: Strength first, Dexterity second, Endurance third (deterministic; no user prompt required).
