# Feature Specification: Navy Character Generator

**Feature Branch**: `001-navy-character-generator`

**Created**: 2026-06-17

**Status**: Draft

**Input**: User description: "We are creating cetools, a suite of tools for the Cepheus Engine tabletop roleplaying game. We are using the Cepheus Engine SRD (https://cepheus-srd.opengamingnetwork.com/) for sourcing the rules of the game. In this first phase of development, we are creating a CLI application for accessing the tools, but the application logic should be decoupled from the CLI logic so we can access the tools in a different manner in the future (for example, using an HTTP API). For the MVP product, we are creating a character generator. The character generator will only support the Navy career in the MVP; however the code for the code generator should be easily extendable to support all additional careers in future development phases. We need to support the full PSEUDO-HEXADECIMAL NOTATION (defined in TABLE: PSEUDO-HEXADECIMAL NOTATION at https://cepheus-srd.opengamingnetwork.com) to support future generation tools."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Generate a Navy Character (Priority: P1)

A game referee or player invokes the character generator and receives a fully-formed Cepheus Engine Navy character, including generated characteristics expressed as a UPP string, career history (terms served, ranks, skills gained), and mustering-out benefits.

**Why this priority**: This is the core deliverable of the MVP. Without a working character generator, no other scenario has value.

**Independent Test**: Invoke the generator with no arguments and verify a complete character record is returned, with a valid 6-character UPP string using pseudo-hexadecimal notation, at least one Navy career term, skills, and benefit results.

**Acceptance Scenarios**:

1. **Given** the tool is invoked, **When** the user requests a new character, **Then** six characteristics (Strength, Dexterity, Endurance, Intelligence, Education, Social Standing) are generated by rolling 2D6 for each, recorded using pseudo-hexadecimal notation as a UPP string (e.g., `687B9C`).
2. **Given** characteristics are generated, **When** the Navy enlistment roll (Endurance 5+) succeeds, **Then** the character undergoes at least one full career term covering: basic training, a survival roll, a commission/advancement attempt, one or more skill rolls, an aging check (if applicable), and a re-enlistment roll.
3. **Given** the Navy enlistment roll fails, **When** the character is drafted or falls back to the Drifter career, **Then** the system records this outcome and the character still completes at least one term before mustering out.
4. **Given** a character has completed all terms, **When** the mustering-out phase runs, **Then** the character receives the correct number of benefit rolls (one per term, plus rank bonuses for O4+), with cash rolls capped at three.
5. **Given** a complete character record, **When** the output is displayed, **Then** the UPP is shown using correct pseudo-hexadecimal notation for all values 0–33.

---

### User Story 2 - View Character as Formatted Output (Priority: P2)

A user receives character output that is human-readable, clearly structured, and suitable for recording on a character sheet or sharing with a game group.

**Why this priority**: Raw data alone is not actionable. Readable output is needed for the tool to be useful at the table.

**Independent Test**: Generate a character and verify the output contains labeled sections for characteristics (UPP), career history, acquired skills, and mustering-out benefits.

**Acceptance Scenarios**:

1. **Given** a generated character, **When** output is displayed, **Then** it includes: the character's UPP string, career title and final rank/title, total terms served, a list of all skills with levels, and all mustering-out benefits received.
2. **Given** a characteristic value above 9, **When** the UPP is displayed, **Then** the value is encoded using the correct pseudo-hexadecimal letter (e.g., 10→A, 14→E, 18→J, skipping I and O).
3. **Given** a character who earned a retirement pension, **When** output is displayed, **Then** the annual pension amount is shown.

---

### User Story 3 - Extend Generator to Additional Careers (Priority: P3)

A developer extending cetools can add a new career by implementing a defined career data structure without modifying the core character generation logic.

**Why this priority**: The extensibility requirement is a stated design goal. While not visible to end users in the MVP, it determines architecture decisions made now.

**Independent Test**: A developer can define a new career (e.g., Scout) using only the data structure interface—without touching the generation engine—and the engine processes it identically to the Navy career.

**Acceptance Scenarios**:

1. **Given** the career data interface is defined, **When** a new career's data is supplied, **Then** the generation engine resolves qualification, survival, commission, advancement, skill rolls, and mustering out using only the supplied data and the shared rules engine.
2. **Given** the core generation logic, **When** it is reviewed, **Then** no Navy-specific logic is hardcoded into the engine; all career variation is expressed through the career data interface.

---

### Edge Cases

- What happens when a survival roll fails? (Character death or mishap, depending on optional rule setting.)
- What happens when a character fails re-enlistment on the first term? (They muster out with only basic training and one skill roll.)
- What happens when a characteristic reaches 0 through aging? (Characteristic is reduced but cannot go below 0.)
- What happens when a characteristic value exceeds 33 (the maximum pseudo-hex value `Z`)? (Values above 33 must still be representable; treat as capped or documented overflow.)
- What happens when a cash benefit roll cap of 3 is already reached? (Subsequent benefit rolls must be taken as material benefits.)
- What happens if Natural 12 is rolled on re-enlistment at 7 terms? (Character must serve an additional term beyond the normal 7-term cap.)

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: The system MUST generate six characteristics (Strength, Dexterity, Endurance, Intelligence, Education, Social Standing) by rolling 2D6 per characteristic.
- **FR-002**: The system MUST represent all characteristic values as pseudo-hexadecimal notation per the full Cepheus Engine SRD table (values 0–33 mapped to `0–9`, `A–H`, `J–N`, `P–Z`, omitting `I` and `O`).
- **FR-003**: The system MUST produce a Universal Personality Profile (UPP) string of six pseudo-hexadecimal characters representing all six characteristics in order: Strength, Dexterity, Endurance, Intelligence, Education, Social Standing.
- **FR-004**: The system MUST resolve Navy career enlistment by rolling 2D6 against the Endurance 5+ target, applying the relevant characteristic modifier.
- **FR-005**: The system MUST process each career term in sequence: basic training (first term only: all Service Skills at Level 0), survival roll (Dexterity 5+), commission roll (Education 6+, Rank 0 only), advancement roll (Education 7+, Rank 1+), one skill roll (two if no commission/advancement track was available), aging check (age 34+), re-enlistment roll (5+).
- **FR-006**: The system MUST apply the correct Navy rank titles (Ensign through Fleet Admiral) and associated bonus skills (e.g., Leadership-1 at Rank 3/Captain).
- **FR-007**: The system MUST allow a character to serve a maximum of 7 terms unless a natural 12 is rolled on re-enlistment, in which case one additional mandatory term is served.
- **FR-008**: The system MUST calculate mustering-out benefits: one roll per term served, plus one additional roll at Rank O4, two at Rank O5, three at Rank O6.
- **FR-009**: The system MUST enforce the cash benefit cap of 3 rolls per character; additional benefit rolls beyond the cap must be taken as material benefits.
- **FR-010**: The system MUST calculate and display the retirement pension for characters who complete 5 or more terms in the Navy (Cr10,000/year at 5 terms, increasing per the SRD schedule).
- **FR-011**: The system MUST apply aging effects to physical characteristics (Strength, Dexterity, Endurance) starting at age 34 (after the fourth term), rolling on the aging table with a negative modifier equal to total terms served.
- **FR-012**: The system MUST assign background skills based on the homeworld education level at character creation (prior to career entry), with at least one skill at Level 0.
- **FR-013**: The character generation logic MUST be implemented as a standalone module or library, independent of any CLI or API delivery layer, so that the same logic can be invoked from a CLI command, an HTTP API endpoint, or any other interface without modification.
- **FR-014**: The career system MUST be implemented via a data-driven interface (e.g., a defined career data structure or schema) so that additional careers can be added without modifying the core generation engine.
- **FR-015**: The CLI MUST expose the character generator as a subcommand (e.g., `cetools character generate`) and print the character record to standard output.

### Key Entities

- **Character**: The generated persona. Carries characteristics, UPP string, age, career history, skill list, and benefit results.
- **Characteristic**: One of six core attributes (Strength, Dexterity, Endurance, Intelligence, Education, Social Standing). Has a numeric value and a derived modifier.
- **UPP (Universal Personality Profile)**: A 6-character string encoding all six characteristics in pseudo-hexadecimal notation.
- **Career**: A structured life path that defines qualification, survival, commission, advancement, re-enlistment target numbers, four skill tables (Personal Development, Service Skills, Specialist Skills, Advanced Education), rank progression with associated titles and bonus skills, and mustering-out tables (cash and material benefits).
- **Term**: A 4-year period of career service. Tracks skills gained and events (survival, commission/advancement outcomes).
- **Skill**: A named capability at a numeric level (0, 1, 2, …). Multiple training rolls in the same skill increase the level by 1 per additional roll.
- **Benefit**: A mustering-out reward, either cash (in Credits) or a material item (passage, weapon, characteristic boost, ship share, etc.).

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: A user can invoke the character generator from the command line and receive a complete, valid Navy character record in under 2 seconds.
- **SC-002**: 100% of generated UPP strings correctly encode all characteristic values using the full pseudo-hexadecimal table (values 0–33), with no use of the omitted letters `I` or `O`.
- **SC-003**: All generated characters satisfy every applicable Cepheus Engine SRD rule for Navy careers (correct target numbers, correct skill table lookups, correct aging thresholds, correct benefit counts) as verified by automated tests covering each rule.
- **SC-004**: A developer can add a second career (e.g., Scout) by defining a career data structure alone, with zero changes to the core generation engine, and the new career produces correctly structured output.
- **SC-005**: The character generation library can be imported and called directly (without invoking the CLI) and returns a structured result usable by any future delivery layer (HTTP API, etc.).

## Assumptions

- The Cepheus Engine SRD is the authoritative rules source; where the SRD is ambiguous, the most common community interpretation will be used and documented.
- The MVP targets single-character generation (one character per invocation); batch generation is out of scope for this phase.
- The optional "mishap instead of death" survival rule is out of scope for the MVP; failed survival rolls result in character death (generation stops and is reported as failed).
- Homeworld background skills are simplified to granting 3 skills at Level 0 chosen from a default homeworld skill list; full homeworld generation is out of scope for the MVP.
- Psionic Strength (the optional 7th UPP digit) is out of scope for the MVP; only the 6-characteristic UPP is generated.
- Anagathics (anti-aging drugs) are out of scope for the MVP.
- The draft system (when Navy enlistment fails) will redirect the character to the Drifter career using the same career data interface, even though Drifter career data is not fully specified in this MVP; the enlistment failure path must be handled gracefully (e.g., report enlistment failure and terminate generation, or enter Drifter with minimal data).
- Output format for the MVP is plain text to standard output; JSON or structured output formats are deferred to a future phase.
- The CLI framework and project packaging are already established via `pyproject.toml` and `uv`.
