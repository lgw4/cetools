# Phase 1 Data Model: World Generation

All types are frozen dataclasses / enums in `src/cetools/engine/worlds/models.py`, following the
project's immutable-value-object convention. Values are the SRD's small integers unless noted.

## Enums

### `TravelZone`

| Member | Code rendered | Meaning |
|--------|---------------|---------|
| `GREEN` | `" "` (space / omitted) | Generally safe (default) |
| `AMBER` | `"A"` | Rule-driven caution flag |
| `RED` | `"R"` | Interdicted—caller-set only (referee discretion) |

### `Density` (subsector world-presence modifier)

| Member | DM | Meaning |
|--------|----|---------|
| `RIFT` | −2 | Rift sector |
| `SPARSE` | −1 | Sparse sector |
| `STANDARD` | 0 | Spiral arm (default, ~50%) |
| `DENSE` | +1 | Densely populated |

## `World`

The primary world of a system. Produced by `generate_world`.

| Field | Type | Range / Rule |
|-------|------|--------------|
| `name` | `str` | Generated by default; caller-overridable; non-empty |
| `size` | `int` | 0–10 |
| `atmosphere` | `int` | 0–15; 0 when `size == 0` |
| `hydrographics` | `int` | 0–10; 0 when `size <= 1` |
| `population` | `int` | 0–10 |
| `government` | `int` | 0–15; 0 when `population == 0` |
| `law_level` | `int` | ≥ 0; 0 when `government == 0` |
| `starport` | `str` | one of `A B C D E X` |
| `tech_level` | `int` | ≥ 0; raised to any applicable SRD minimum |
| `population_modifier` | `int` | 1–10 when `population > 0`, else 0 |
| `trade_codes` | `tuple[str, ...]` | zero or more 2-letter codes, in table order |
| `travel_zone` | `TravelZone` | `AMBER` if the Amber rule matches, else `GREEN` (or caller-set `RED`) |

**Derived (methods/properties, no stored state):**

- `profile` → classic UWP string: `starport + pseudohex(size,atmosphere,hydrographics,population,
  government,law_level) + "-" + pseudohex(tech_level)` (e.g. `A867A9C-F`).
- `head_count` → `population_modifier * 10**population` (0 when population 0).

**Invariants** (validated in tests, enforced by the generator):

- SC-001 ranges hold for every field.
- SC-002 dependencies: `size==0 ⇒ atmosphere==0 and hydrographics==0`;
  `size<=1 ⇒ hydrographics==0`; `population==0 ⇒ government==law_level==tech_level==0`;
  `tech_level` ≥ every matching minimum.

## `System`

A world plus its stellar surroundings. Produced by `generate_system`; each subsector hex holds one.

| Field | Type | Range / Rule |
|-------|------|--------------|
| `world` | `World` | the primary world |
| `hex` | `str \| None` | `"CCRR"` (column, row) within a subsector; `None` for a standalone system |
| `planetoid_belts` | `int` | ≥ 0; ≥ 1 present when world `size == 0` |
| `gas_giants` | `int` | ≥ 0 |
| `naval_base` | `bool` | per SRD; only possible on starport A/B |
| `scout_base` | `bool` | per SRD; never on starport E/X |
| `pirate_base` | `bool` | per SRD; never with starport A or a naval base |
| `allegiance` | `str` | default `"Na"` |

**Derived:**

- `base_code` → `A` (naval+scout), `N` (naval), `S` (scout), `G` (scout+pirate), `P` (pirate), else
  `" "`.
- `pbg` → 3-digit string: `population_modifier`, `planetoid_belts`, `gas_giants`.
- `data_line` → full world-data line (see `profile.py` / research D5).

**Invariants:** base-exclusion rules never violated (SC-004); `world.size==0 ⇒ planetoid_belts>=1`.

## `Subsector`

An 8×10 region. Produced by `generate_subsector`.

| Field | Type | Rule |
|-------|------|------|
| `systems` | `tuple[System, ...]` | one per occupied hex; each `.hex` set to its `"CCRR"` |
| `density` | `Density` | the density used to generate it |

**Invariants:**

- Every `system.hex` is within columns 01–08, rows 01–10, and is unique.
- Occupied share ≈ 50% at `STANDARD`, shifting with density (SC-007).
- All auto-generated `system.world.name` values are unique within the subsector (SC-008).

## Static tables (`worlds/tables.py`)—data, not logic

| Name | Shape | Purpose |
|------|-------|---------|
| `SIZE_DESCRIPTIONS` | `tuple[str, ...]` indexed by size | diameter/gravity notes (for future rich output) |
| `ATMOSPHERE_DESCRIPTIONS` | `tuple[str, ...]` | atmosphere labels |
| `HYDRO_DM_BY_ATMOSPHERE` | `dict[int, int]` | −4 / −2 DMs |
| `POPULATION_DMS` | data predicates | the five population DM rules |
| `STARPORT_BY_ROLL` | `dict[int, str]` / list | Primary Starport table (clamped ≤2, ≥11) |
| `GOVERNMENT_TYPES` | `tuple[str, ...]` | government labels (0–15) |
| `LAW_LEVELS` | `tuple[str, ...]` | law labels |
| `TL_DM_BY_VALUE` | `dict[str, dict[int, int]]` | TL DM matrix keyed by column then value |
| `TL_MINIMUMS` | list of `(predicate-data, min)` | TL minimum overrides |
| `TRADE_CODES` | list of `(code, field-range-set)` | trade classification conditions |

Trade-code and population-DM conditions are expressed as per-field allowed-value sets so the matcher
iterates uniformly (no per-code branching in the generator).

## Relationships

```text
Subsector 1 ──* System 1 ──1 World
                    │
                    └─ bases (naval/scout/pirate) → base_code
World ── trade_codes (derived from own UWP)
World ── travel_zone (AMBER derived; RED caller-set)
```

## Generation order (dependencies)

`size → atmosphere → hydrographics → population → government → law_level → starport → tech_level →
population_modifier → trade_codes → travel_zone` (World), then `planetoid_belts, gas_giants, bases,
allegiance` (System), then presence + naming-uniqueness (Subsector). Matches research Part A.
